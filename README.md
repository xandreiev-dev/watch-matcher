# Watch Matcher

Приложение для обработки XLSX-файлов с объявлениями о смарт-часах.

Позволяет:
- извлекать бренд, модель, цвет и гарантию
- сопоставлять модели с эталонными данными
- отображать результат в виде таблицы

## Принятые решения
1. Backend и frontend разделены
2. Логика backend разбита на сервисы:
 - обработка Excel
 - извлечение данных
 - получение эталонных моделей
 - сопоставление
3. Используется API /vendor/watch-models для получения эталона
4. Добавлена нормализация строк для повышения качества сопоставления
5. Используется fallback-логика для извлечения гарантии
6. Использован сигнатурный подход к сопоставлению вместо fuzzy matching

## Покрытые кейсы
1. загрузка XLSX-файла
2. обработка XLSX по ссылке
3. извлечение бренда из названия
4. извлечение модели с учётом шумных данных
5. извлечение цвета (ru/en)
6. извлечение гарантии (включая частичные данные)
7. сопоставление моделей:
 - точное совпадение нормализованных строк
 - сопоставление по сигнатурам (brand, family, generation, variant)
 - ограниченный fallback для слабых моделей
8. обработка мусора в названиях
9. обработка аксессуаров и нерелевантных товаров

## Ограничения
1. качество зависит от входных данных
2. не все редкие модели могут быть сопоставлены
3. извлечение гарантии эвристическое
4. возможны случаи Unknown при недостатке информации

## Подход к сопоставлению моделей

В проекте реализован *сигнатурный подход к сопоставлению моделей*, вместо классического fuzzy matching.

### Почему не fuzzy matching?

Fuzzy matching часто даёт некорректные результаты, особенно в случаях:
- разные поколения (Watch 6 vs Watch 7)
- разные версии (Pro / Classic / Ultra)
- аксессуары ошибочно сопоставляются как часы

### Реализованное решение

Каждая модель преобразуется в структурированную **сигнатуру**:

(brand, family, generation, variant)

Примеры:
- Samsung Galaxy Watch 6 Classic → ("samsung", "watch", "6", "classic")
- Apple Watch Series 11 → ("apple", "series", "11")
- Huawei Watch Fit 4 Pro → ("huawei", "fit", "4", "pro")

### Стратегия сопоставления

1. Точное совпадение нормализованных строк
2. Сопоставление по сигнатурам:
   - строгое сравнение:
     - бренда
     - семейства
     - поколения
     - варианта
3. Ограниченный fallback:
   - только для слабых (общих) моделей
   - не переопределяет точные совпадения

### Основные принципы

- Не выполняется сопоставление для `Unknown` бренда
- Аксессуары (ремешки, браслеты и т.д.) не сопоставляются как часы
- Разные версии не схлопываются (Pro ≠ базовая модель)
- Разные поколения не смешиваются (Watch 6 ≠ Watch 7)
- Приоритет: **точность > количество совпадений**

## Ограничения и допущения

- Некоторые модели остаются `Unknown` намеренно, чтобы избежать ложных совпадений
- Версии моделей (Pro / Classic / Ultra) обрабатываются строго
- Если модель отсутствует в эталоне (например Band / Kids), она остаётся unmatched

## Итог

Решение ориентировано на:
- структурированное извлечение данных
- детерминированную логику сопоставления
- минимизацию ложных совпадений

Вместо максимального количества совпадений приоритет отдаётся их корректности и объяснимости.
