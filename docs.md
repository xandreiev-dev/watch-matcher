# Documentation

## Архитектура

Проект разделён на два слоя:

- Backend (FastAPI)
- Frontend (Vue 3)

---

## Backend

Основные компоненты:

- `excel_service`  
  Чтение XLSX и валидация структуры

- `extraction_service`  
  Извлечение:
  - бренд
  - модель
  - цвет
  - гарантия

- `vendor_service`  
  Получение эталонных моделей через API:

/vendor/watch-models


- `matching_service`  
Сопоставление моделей:
- нормализация строк
- удаление мусора
- fuzzy / contains / exact matching

---

## Frontend

Компоненты:

- `FileUpload.vue`  
Загрузка файла или ввод URL

- `ResultsTable.vue`  
Отображение результата

- `api.js`  
Работа с backend API

---

## Поток данных

1. Пользователь загружает файл или вводит URL  
2. Frontend отправляет данные на backend  
3. Backend:
 - читает Excel
 - извлекает данные
 - получает эталон
 - сопоставляет модели  
4. Результат возвращается на frontend  
5. Таблица отображается пользователю  

---

## Формат результата

"Название": "...",
"Бренд": "...",
"Модель": "...",
"g_model_matched": "...",
"Цвет": "...",
"Гарантия": "...",
"image_url": "...",

## Особенности
- поддержка RU и EN данных
- устойчивость к шуму в тексте
- fallback логика для неполных данных
